Size & Typography

_heart.scss

Sizing, typography, and baseline are all interconnected.

Sizes

The heart contains units, sizes and other base definitions. All the spacing, typography, and sizing is derived out of a base sizing unit: $size: 16px. 16px is chosen as it's equivalent to 1 rem, and is easily divisible by 2, 4, and 8.

16px type is comfortably readable size—not too big nor not too small.

The following boxes illustrate the other standard available sizes. They're roughly built with a golden ratio typographic scale.

m4 means $base minus 4 levels, while p4 means $base plus 4 levels. Not perfect but flexible enough for real-world use.

Baseline

Baseline and vertical rhythm are very important for print design, and increasingly important for site design. Having a solidly established vertical rhythm allows for more complex designs, as all components end up aligning to a grid.

With that said, establishing a vertical rhythm for a website, especially a dynamic site where copy and images can be of variable height, is still difficult.

  • Base font size is set to 16px as default.
  • Base line height is set to 16px * 1.5 or 24px as default.
  • Baseline is based on 8px line-height ($size-base/2) for finer control; all heights should try to adhere to a multiple of 8, like 16px, 24px, 32px, or 40px heights.
  • Padding, and margins are always created using $size-base (usually 16) or half or double that value. Ideally all objects should have a baseline of 16 pixels, though this is almost never possible.
  • Line heights are usually established at the font size * 1.5 but may need to be tweaked depending on the typeface.
  • Links and other inline elements attempt to flow with the copy at 16px or the appropriate line height (default to 24px)
  • Elements like form inputs, buttons and so on will need to contain copy at 16px line-height, so these are naturally going to be larger.

Links

Links are styled by default with no default browser underline, but with a border-bottom. Borders can be animated with keyframes and are more flexible, but can run into wrapping and other issues (if a link is wrapped around text, the border won't render).

Code

Code blocks and inline code are pretty straight-forward. Pygments zenburn is used as the default code theme. Examples can be found all throughout stylecoeur.

  • .highlight is used to specify example code
